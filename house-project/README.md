# План этажа — интерактивный редактор (2 этажа)

Автономное браузерное приложение для предварительной оцифровки поэтажных планов
дома. Работает без сервера и без сборки. Поддерживает **два этажа** —
F1 (первый) и F2 (второй) — с независимой геометрией и историей правок.

## Запуск

Двойной клик по `index.html`. При первом открытии оба этажа автоматически
подгружаются из встроенного `source-plans.js` (копии исходных
`input/first-floor-layout.json` и `input/second-floor-layout.json` — оригиналы
в `input/` при работе редактора НЕ изменяются).

## Переключение этажей

- Кнопки **«1» / «2»** в верхней панели.
- Горячие клавиши: **1** — первый этаж, **2** — второй.
- Каждый этаж имеет собственные:
  - геометрию (вершины, стены, помещения, двери, окна, лестница, мебель);
  - историю undo/redo;
  - масштаб и положение вьюпорта.

## Что уже размечено (по обеим фотографиям)

### Первый этаж (F1)
- 6 помещений: **R01** большая левая зона (гостиная?), **R02** правый верхний
  холл с лестницей, **R03** средняя правая, **R04** нижняя правая,
  **R05** нижний центральный выступ (тамбур/крыльцо), **R06** центральная
  проходная зона.
- 20 стен `F1-W01…F1-W20` (наружные / перегородки / «interface» — граница
  выступа).
- 5 дверей `F1-D01…F1-D05`.
- 8 окон `F1-O01…F1-O08`.
- 1 лестница `F1-S01` в R02.

### Второй этаж (F2)
- 8 помещений: **R01–R03** левая колонна (три спальни / кабинета),
  **R04** лестничный холл, **R05** средняя правая, **R06** нижняя правая,
  **R07** центральный коридор, **R08** нижний центральный выступ
  (балкон/веранда).
- 24 стены `F2-W01…F2-W24`.
- 6 дверей `F2-D01…F2-D06`.
- 9 окон `F2-O01…F2-O09`.
- 1 лестница `F2-S01` в R04.

Все размеры кроме координат вершин помечены статусом `estimated`. Ни один
размер **не подтверждён замером**.

Исходные координаты source-JSON (нормализованные 0–1000) конвертируются в мм
с коэффициентом **1 ед. = 10 мм**, что даёт габарит ~10 × 10 м. Настоящий
масштаб нужно откалибровать по одному подтверждённому размеру (см. ниже).

## Что сомнительно

- Назначение всех помещений (кроме лестничных зон).
- Ширина коридора и точные положения дверей.
- Толщины стен приняты по умолчанию: 400 мм наружные, 150 мм перегородки.
- Все высоты — `unknown`.
- Стены типа `interface` (граница нижнего выступа с основным объёмом)
  сохранены как перегородки с примечанием.
- Внешний контур первого и второго этажей визуально совпадает, но требует
  точного сверения.

## Порядок ввода реальных размеров

1. Задать **один эталонный размер** (например, длину наружной стены). После
   ввода «заявленной длины» со статусом `entered`/`verified` соответствующая
   вершина будет пересчитана — потом можно вручную довести остальные.
2. **Наружные размеры** обоих этажей (они должны совпасть).
3. **Размер нижнего центрального выступа**.
4. **Толщины наружных / внутренних стен**.
5. **Высоты потолков** и **стен**.
6. **Ширина и положение каждой двери и окна**.
7. **Размеры лестницы** (ширина марша, глубина, направление подъёма).
8. **Подтверждённое назначение помещений**.

## Интерфейс

### Верхняя панель
- Кнопки этажа **1 / 2**.
- **Показать весь план** — вписать в экран.
- **Отменить / Повторить** (Ctrl+Z / Ctrl+Y) — независимо на этаж.
- **Импорт JSON** — принимает как один этаж (в схеме редактора), так и
  проект `{floors: {F1, F2}}`.
- **Экспорт этажа** — сохраняет только текущий этаж (`plan-f1.json` /
  `plan-f2.json`).
- **Экспорт проекта** — сохраняет оба этажа в одном файле
  `plan-project.json`.
- **Сброс** — возвращает оба этажа к исходному размеченному состоянию
  (с подтверждением).

### Слои (левая панель)
Включение/выключение номеров помещений и стен, размеров, дверей, окон, мебели,
лестницы.

### Добавление
- **+ Стена**: два клика задают концы. Если клик попадает в существующую
  вершину, она переиспользуется (сохраняется связность).
- **+ Помещение**: последовательно кликайте стены по контуру, затем `Enter`.
- **+ Дверь / + Окно**: клик по стене размещает проём.
- **Мебель**: кнопки каталога — кровать, диван, шкаф, стол, стул, тумба,
  кухня, сантехника. Далее перетаскивание мышью, поворот через инспектор.

### Инспектор (правая панель)
- Стена: тип, заявленная длина (со статусом), толщина, высота, примечание.
- Помещение: название, назначение, высота потолка, авт. площадь, список стен.
- Дверь: родительская стена, расстояние от начала, ширина, высота, петли,
  открывание.
- Окно: родительская стена, расстояние, ширина, высота, высота подоконника.
- Мебель: тип, габариты, поворот, координаты, помещение.
- Лестница: габариты, число маршей, направление.

Каждый размер имеет один из статусов и отображается на плане своим цветом:
- серый — `unknown`;
- жёлтый — `estimated`;
- оливковый — `entered`;
- зелёный — `verified`;
- красный — `conflict`.

### Списки (правая панель)
- Все элементы с неизвестными размерами — клик переходит к объекту.
- Предупреждения геометрии: незамкнутые контуры, дверь/окно за пределами
  стены, конфликт длины, ошибки JSON.

| Клавиша | Действие |
|---------|----------|
| `1` / `2` | Переключение этажей |
| `Ctrl+Z` / `Cmd+Z` | Отменить |

## Snap по Shift (как в Photoshop / Figma)

- **Shift при перетаскивании вершины** — снап направления от исходной точки
  под углы, кратные 45°. То есть можно тянуть строго вправо / вниз / по
  диагонали, соседние стены сохраняют угол.
- **Shift при перетаскивании мебели / лестницы** — блокировка по большей оси:
  движение либо только по X, либо только по Y.
- **Shift при добавлении стены** (когда первая точка уже поставлена) — снап
  второй точки под 45°-сетку. Пока Shift зажат, предварительная линия рисуется
  зелёным; без Shift — синим (свободный угол).

Shift можно зажимать и отпускать в любой момент во время перетаскивания —
поведение переключится сразу.

## Формат JSON

### Один этаж (внутренний формат редактора)

```json
{
  "schemaVersion": "2.0",
  "units": "mm",
  "sourceMeta": { "floor": 1, "image": "first_floor.jpg" },
  "vertices": [{ "id":"v001", "x":450, "y":900 }],
  "walls": [{
    "id":"F1-W01", "v1":"v001", "v2":"v002", "type":"exterior",
    "thickness":     { "value":400, "status":"estimated", "source":"image", "note":"" },
    "height":        { "value":null,"status":"unknown","source":"image","note":"" },
    "declaredLength":{ "value":null,"status":"unknown","source":"image","note":"" },
    "note":""
  }],
  "rooms": [{ "id":"F1-R01","name":"...","walls":["F1-W01","..."], "..." }],
  "doors":    [{ "id":"F1-D01","wallId":"F1-W02","distance":{...},"..." }],
  "windows":  [{ "id":"F1-O01","wallId":"F1-W01","..." }],
  "stairs":   [{ "id":"F1-S01","roomId":"F1-R02","x":..,"y":..,"..." }],
  "furniture":[]
}
```

### Проект (оба этажа)

```json
{
  "version": "3.0",
  "exportedFloors": ["F1","F2"],
  "floors": {
    "F1": { /* формат этажа */ },
    "F2": { /* формат этажа */ }
  }
}
```

## Задел на 3D

Данные достаточны для построения 3D-модели двухэтажного дома:
- каждая стена — две вершины + толщина + высота;
- каждый проём — положение вдоль стены + вертикальные размеры;
- каждая лестница — координаты и направление, потенциально связывает этажи;
- каждый предмет мебели — координаты, поворот, габариты.

Следующий шаг: `three.js` (или аналог) использует тот же `plan-project.json`
и строит два этажа с реальной толщиной перекрытия между ними.
